## 4. 结果评估指南

### 4.1 准确性评估（Accuracy）

**评估方法**：
1. 随机抽取初稿中3-5个陈述
2. 对照原始来源验证是否一致
3. 记录不一致或疑似幻觉的内容

**评分标准**：
- 5分：所有陈述与来源完全一致
- 4分：1处轻微不一致
- 3分：2-3处不一致
- 2分：大量不一致
- 1分：基本不可用

### 4.2 可追溯性评估（Traceability）

**评估指标**：
- 溯源覆盖率 = 有来源标注的陈述数 / 总陈述数 × 100%
- 引用正确性 = 正确溯源的引用数 / 总引用数 × 100%

**评分标准**：
- 5分：覆盖率100%，引用100%正确
- 4分：覆盖率>=80%，引用>=90%正确
- 3分：覆盖率>=60%，引用>=80%正确
- 2分：覆盖率<60%
- 1分：几乎无法追溯

### 4.3 检索质量评估

**评估指标**：
- 召回率：检索结果中相关文档的比例
- 精确率：相关文档在所有检索结果中的比例
- MRR（平均排名倒数）：首个相关结果排名的倒数

**评分标准**：
- 相关性>=4分的结果占比>80%为优秀
- 相关性>=3分的结果占比>60%为合格

## 5. 常见问题

### Q1: 运行报错 "Module not found"

**解决**：
```bash
pip install -r requirements.txt
```

### Q2: transformer.pdf 下载失败

**解决**：手动下载
1. 访问 https://arxiv.org/pdf/1706.03762
2. 另存为 `D:\Users\作业\科研实践\memory\raw\papers\transformer.pdf`

### Q3: ChromaDB报错 "Permission denied"

**解决**：
```bash
# 关闭可能占用ChromaDB的Python进程
taskkill /f /im python.exe

# 或删除chroma目录重新开始
rm -rf "D:\Users\作业\科研实践\memory\chroma"
```

### Q4: 中文显示乱码

**原因**：Windows终端编码问题

**解决**：演示脚本会自动处理，不影响结果保存

### Q5: Mock LLM响应不符合预期

**说明**：Mock模式下使用预设响应，不依赖真实LLM。如需真实响应：
1. 安装并启动 Ollama
2. 修改 `config.yaml` 中 `llm.provider: "ollama"`
3. 重新运行 `python demo.py`
